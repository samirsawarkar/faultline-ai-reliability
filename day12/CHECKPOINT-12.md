# CHECKPOINT-12 — fault catalog + gallery

**Mission.** Consolidate every fault into a defensible, reproducible experimental
specification.

**Fail condition.** Any fault card lacks trigger, trace, detector, recovery, or
metric. — **Not triggered.**

## What was built

The six-fault spectrum (F1–F6, Days 8–11) consolidated into one catalog:

- **Six complete fault cards** — each with a normalized 10-field spec plus the five
  required fields (trigger, trace, detector, recovery, metric), enforced by
  `FaultCard.validate()`.
- **A gallery** — `catalog.json`, `GALLERY.md`, and a static `catalog.html` linking
  each fault → trace → detector → metric.
- **Labelled traces** — one canonical, gapless trace per fault under
  `evidence/traces/`, regenerated by the owning day + the Day-4 tracer.
- **An audit** — reproducibility + ground-truth integrity + completeness.
- **A taxonomy** — faults grouped by producing component.

12 tests gate it; seven evidence artifacts (plus six traces) prove it.

## The catalog

| Fault | Producing component | Detector (nature) | Recall | Recovery |
|---|---|---|---|---|
| F1 structured-output corruption | tool output | schema (deterministic) | 1.0 (0.667 @sev1) | repair + retry |
| F2 latency / timeout | provider transport | duration vs budget (deterministic) | 1.0 | timeout + retry |
| F3 schema drift / wrong data | tool output | schema + invariant (mixed) | 0.6 | semantic validation |
| F4 provider error | provider transport | explicit flag (deterministic) | 1.0 | circuit breaker + fallback |
| F5 context corruption | agent context | consistency invariant (semantic) | 0.5 | context re-grounding |
| F6 loop exhaustion | agent control loop | repetition + budget (deterministic) | 1.0 | step-budget cap |

Every recall is pulled live from the day that measured it (Days 9–11), so the
catalog cannot drift from the evidence.

## Audit — PASSED

`evidence/audit_report.json`:

- **all_reproducible: true** — each fault's canonical trace regenerates
  byte-identically; the whole catalog rebuilds identically (verified across hash
  seeds).
- **no_label_leakage: true** — no fault's ground-truth labels/ids appear in its
  observable trace, across all six.
- **all_cards_complete: true** — every card has trigger, trace, detector,
  recovery, metric.
- **all_detectors_scored_vs_truth: true** — every metric is a real number measured
  against the injection log, tagged with its source day.

## Failure taxonomy by producing component

- **Provider boundary** (F1–F4): content faults (F1 malformed, F3 semantic wrong)
  + transport faults (F2 timing, F4 availability).
- **Agent internals**: F5 (context/state, semantic) + F6 (control loop,
  deterministic).

The pattern: transport, control, and shape are deterministic to detect; value
correctness (data plane + context) is semantic — value faults come from the two
components that hold values.

## Mastery gate — all five

- **Explain** — `LEARN-taxonomy.md`: the producer-first taxonomy and why it
  predicts the hard cases.
- **Build** — `faultline_catalog/`: cards, catalog, gallery, taxonomy, audit.
- **Debug** — `evidence/traces/*.json`: one labelled, gapless trace per fault.
- **Measure** — `evidence/catalog.json` (live metrics) + `audit_report.json`.
- **Defend** — `DECISIONS.md` (D12-001…D12-007); completeness is enforced by a
  validator, metrics are pulled live, and the audit ships as evidence.
